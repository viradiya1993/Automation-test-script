AHQ Local Agent

AHQ Local Agent is a client side application of AutomationHQ that is installed on the user machine. This agent coordinate the test execution with the AutomationHQ web application. It is used for both debugging (dry run) of Test Script while development phase and also to run Test Bot on local machine in silent mode.

Workflow

  • Download AHQ Agent
  • Install AHQ Agent
  • Download AHQ Test Script Agent
  • Download AHQ Test Bot Agent
  • Start / Stop AHQ Local Agent

Download & Install AHQ Agent

You can download AHQ Agent from the onboarding wizard during onboarding or from the AHQ Home Page

  • Go to AHQ Home Page and click on Download AHQ Agent based on your local machine operating system
  • On the windows machine, double click on exe downloaded in your /download folder
  • Click on Show More Info and Click on Run anyway for the “Widows protected for your PC” message
  • Once you complete installation then you can see AHQ Agent in your windows start menu

Download AHQ Test Script Agent

AHQ Test Script Agent is for running the test script in local machine for the debugging purpose. When you are creating test script in AHQ, you want to verify if the script is running fine and if its not then where is it failing. This debugging can be done in Test Script module with the help of this AHQ Test Script Agent on your local machine.

  • Launch AHQ Agent from windows start menu
  • Please check the Java JDK version on your machine and if not available then download as instructed on the AHQ Agent
  • In the Test Script tab click on Download link. It will download a file and once its complete then it shows Start - Stop buttons.
  • Click on Start button to start the local agent. It will give a successful start message in the console when it’s ready.
  • Your machine is now ready for running test script in debug mode from AutomationHQ web app

Download AHQ Test Bot Agent

AHQ Test Bot Agent is for running the test bots in local machine for free community edition of AutomationHQ. If you are using AHQ community edition then running test bot on local machine is the default option and before you run your test bot, you must start this AHQ Test Bot Agent on your local machine.

  • Launch AHQ Agent from windows start menu
  • In the Test Bot tab click on Download link. It will download a file and once its complete then it shows Start - Stop buttons.
  • Click on Start button to start the local agent. It will give a successful start message when it’s ready.
  • Your machine is now ready for running test bots from AutomationHQ web app

Element Library

In this module you can enter the details of the web & mobile elements on which you want to develop the test scripts.

Workflow

  • Add Application
  • Add Page
  • Add Elements

Add Application

This is the application under test. If you have multiple web applications then add the name of your application to uniquely identify each app.

  • Click on Add Application button
  • Enter name of your application in Application Name
  • Enter the URL of the web address from where you wanted to record the elements in URL field
  • Click SAVE button to save the application

Add Page

This is a web page for the web application. Each web page should be recorded separately for the easy maintenance.

  • Select the Application Name under which you want to add page.
  • Click on Add Page button
  • Enter name of the page in Name
  • Enter the description of the page in Description field
  • Click SAVE button to save the page

Add Elements

Element or Locators are the component of your application that automation engine can find for doing any action. You should name it appropriately to easily identify them on the page and in test scripts.

  • Select the Application and select the page
  • Click on Add Element button
  • Select from drop down list which Type of element you are adding
  • Select appropriate By based on which parameter of element you are recording
  • Enter the value of the element parameter in Value field
  • Click SAVE button of the page to save the elements

Test Script

In this module you can create your test automation scripts without any coding.

Workflow

  • Add Epic
  • Add Story
  • Add Test Script
  • Add Test Steps
  • Add Test Data
  • Test Script Debug Run

Add Epic

Epic can be same Epic that you have in your agile sprint planning or you can enter the major module of your application as Epic.

  • Click on Add Epics button
  • Enter name of your Epic in Name
  • Enter the details of the Epic in Summary field
  • Click SAVE button to save the Epic

Add Story

Story can be same Story that you have in your agile sprint planning or you can enter the minor features of your application as a story.

  • Select the Epic under which you want to create a story. (you can also create story without selecting Epic, in that case you can select Epic from the Select Epic field )
  • Click on Add Story button on top right menu
  • Enter name of the story in Story Name field
  • Click SAVE button to save the story

Add Test Script

Test Script is the basic test case that you wanted to execute and verify specific point in your application.

  • Select the Epic under which you want to create the test script. (you can also create test script without selecting Epic & Story, in that case you can select Story from the Select Story field )
  • Expand the Story Name under which you wanted to create test script
  • Click on Add Test Script button under this story name (Test Script creation page will open up)
  • Enter name of the test script in Name field
  • Enter description of the test script in Description field. You can also enter the manual test steps in this field so that later on you can refer this section for automation script.
  • Select status of the test script from Status dropdown list
  • Select which application this test script belongs to by selecting name from Application field. If you haven’t created any application then you can use Add Application link here to create new application.
  • You can enter the feature name of your application in the Tag field. This field will be helpful in Analytics to review test scripts status based on your application features.

Add Test Steps

These are pre-defined test steps that is available for creating new test script.

  • Enter the text in Search box of the test steps. Test steps list will be filtered based on the text you search.
  • Select the test step that you would like to add to your test script and drag and drop in the left side of the box
  • Enter the value or select locator in the selected test step. If you want to add flexible value then you can create variable in Settings à Global Parameters à Custom Properties and put the config[variableName] as value here in the test step. For the Environment variable you can use config.baseUrl which will help selecting dynamic environment during run time
  • You can click Save button from top menu to save the test script

Add Test Data

If you want to create data driven test scripts so that you can parse multiple data to your test script then you can create test data table in the test script module. When you run the test script then every time it will take data from one row of the data table then execute all test steps for that data.

  • Open Test Script if its closed
  • In the selected test step where you want to put the data instead of fixed value then type data.[field-name] and press enter key on keyboard. You can see the [field-name] column appears in the data table
  • Click on Add Row button to add one row in the data table
  • Enter the value in each cell and save by clicking check mark button
  • Alternatively you can import the csv file for the data from Import button
  • Click on Save button from top menu on test script to save test script

Add Test Script Debug

You can run your test script on your local machine to check if the test script you created is working as expected. You can only run one test script at a time in debug mode.

  • Launch AHQ Local Agent on your system
  • Go to Test Script and click on Start Button. Wait until the message on console shows successful start.
  • In your AutomationHQ web app, open Test Script if its closed
  • Click on Save & Run button. Alternatively you can click Play button icon beside the test script in the test script list
  • This will launch a debug page for the test script with all test steps listed in it.
  • Click on Run button. It will run each test step on your local machine and you can review the test run.
  • If you want to edit the test script then you can click on Edit button.

Test Suite

In this module you can group test scripts together based on various criteria of grouping the test scripts e.g. Feature based grouping, Execution based grouping.

Workflow

  • Add Test Suite
  • Add Test Scripts

Add Test Suite

Test Suite is a group of test scripts bound together based on the purpose. Test Suite can be further used in the test execution area as part of Test Bot.

  • Click on Add Test Suite button
  • Enter name of the test suite in Name field
  • Enter description of the test suite in Description field. You can put details of the purpose of test suite and what it contains to easily identify the groupings later.

Add Test Scripts

  • Select Test Suite Name in which you wanted to add test scripts
  • Select the Test Script from the right side box, drag & drop it to left box to associate test scripts to this test suite
  • Alternatively you can start entering name in the Search box. You will see list of Epics, Stories, Test Scripts appear in the drop down list
  • Select any Epic or Story from the drop down list and it will filter all test scripts in right side box and only test scripts available that are associated with that Epic or Story. You can than drag and drop as selection in left box. If you select the test script from the search drop down list then it will directly get selected and appears in the left box.
  • Click on Save button to save the test scripts association with Test Suite.

Test Bot

Test Bots are the virtual testing robots in AutomationHQ. You can configure them with the information of what test suites to run, where to run, when to run and it will run the automation test scripts based on these instructions.

Workflow

  • Add Test Bot
  • Edit Testbot details
  • Execute your Test Bot
  • Schedule Test Bot

Add Test Bot

TestBot is a batch run where you can run 100s of scripts at the press of a button and it executes all test in silent mode.

  • Click on Test Bot button on top left menu.
  • Enter name of the Test Bot in Name field.
  • Enter the details of the Test Bot in Summary field.
  • Click SAVE button to save the Test Bot.

Edit Test Bot Details

You can provide all the details related to test bot in this detail tab.

  • Select your Test Bot form the left menu.
  • Select Test Bot Details.
  • In the Test Suites section, Drag and drop the Test Suites you want to run from the right panel.
  • Click on Save to save your Test Bot details.

Execute your Test Bot

You can run your test Bot on your local machine or remotely (only available if you are paid client) then it runs on our remote server and shows the recorded video as well in results.

  • Launch AHQ Local Agent on your system.
  • Go to Test Bot and click on Start Button. Wait until the initiating status changes to Running.
  • In the AHQ web app, enter the name of Test run in the Name field.
  • Select your test environment from the Environment dropdown (This is where your Test Bot will run)
  • If you want to add a new Environment Click on Add Environment where you can add your Environment name in the Name field and your Environment URL in the URL field then click Save to save your Environment.
  • Add Time out in the Time out field which is the number of seconds AHQ will wait if the current command is not returning pass or fail value.
  • Add Time out in the Time out field which is the number of seconds AHQ will wait if the current command is not returning pass or fail value.
  • Check or Uncheck Screenshot on Error as needed, it provides you with a screen shot of each error.
  • Check or Uncheck Screenshot on Finish as needed, it provides you with a screen shot after finishing the Execution.
  • Check or Uncheck Close Browser after Each Execution as needed, it closes the browser after each execution.

For Premium Users

  • You can select Type, Browser, Browser Version, OS Type and Resolution.
  • For Grid URL if you select the Grid as "AHQ Local Agent" then test will run on the local machine in a silent mode. If you select Grid as "AHQ Premium Grid" then it will run on our remote server and shows the recorded video as well in results.
  • Check or Uncheck Screenshot After Each Step as needed, it provides you with a screen shot of each test step
  • Scheduler tab will be enabled for the Premium user to schedule the Test bot run on daily, weekly, monthly basis on specific time.

Test Results

  • Select Test Bot Name
  • In right panel click and expand the Test Results (only available if you have executed this test bot in past)
  • Click on Test Suite
  • Click on Test Script
  • You can review test results of each test step in this window
  • Click on Edit button on top to edit this test script in Test Script module

Projects and Users

In this module you can add the details of your Projects & and mange their users , each Project can has it's own Elements Library , Test Scripts , Test Suites , Test Bots , Analytics , Settings and Users. Projects are either enabled or diabled , enabled projects appear in the Projects drop down list at the top of the page and diabled projects dosen't appear in this list .

Workflow

  • Add Project
  • Invite Users
  • Assign Users to a certain project

Invite Users

  • Select All Users from the right panel.
  • Click on Invite Users button.
  • Enter the Email addresses in the Email Address field of invited users , Press Enter after each email if you want to add another one .
  • Click on Send.
  • Now an Invitation email is sent to the user , when the user accepts the invitation and complete the Regestration steps ,The status will be changed from Invited to Enabled,also you can click on the circular arrow to Resent invitation to the user.
  • If you want to delete a user , Click on the Bin Icon then Yes , so you will get user deleted.
  • If you want to disable an enabled the user , click on the slide toogle , so the Status will be changed from Enabled to Disabled.

Add Project

You can add different projects to work on ,and diable or enable them.

  • Click on Add Project button.
  • Enter Project name in the Project Name field
  • Enter Project Description in the Project Description field
  • If you want to enable the user leave the slide toggle as purple , if you want to disable the project click on the slide toggle so it will be gray, enabled projects appear in the Projects drop down list at the top of the page and disabled projects doesn't appear in this list.

Assign Users to Certain project

You can assign users to your project each user will have a different view according to his Role , there are four roles in AHQ.

  • Project Admin: where User can create new Project, invite/assign Users, disable/remove Users, setup test environment, global parameters
  • QA Director: where User can access Analytics Module for all project in an organisation and drill to Test Results.
  • Tester: where User can create test script, Test Suites, TestBots for execution, can execute test on local machine.
  • QA Manger: whereUser can access Analytics Module for assigned project in an organisation and drill to Test Results.
  • Select the project you want to assign users to from All Projects panel at the left.
  • Click on Assign User button at the top right panel.
  • Enter the User name in the User field , and select the desired user from the drop down list, you can add many users by pressing enter between each one .
  • Select the Role of the user from the Role field , Assigned users gets view according to the role assigned to them .
  • Click on Save button to save your assigned users.
  • Assigned users appear at the bottom of Assigned Users panel.

Settings

In this module you can setup various parameters and settings for your project.

Workflow

  • Environments
  • Global Parameters

Environments

This is the area where you can define various environment you have for your applications e.g. Test Environment, Pre-PROD environment. Purpose of creating these environments is to make your test scripts flexible to run it on any environment. In order to use this flexibility, you have to use config.baseUrl when you put the URI of your application in test script.

  • Click on Settings
  • Click on Environments
  • Click on Add Environment
  • Add Name of your environment in Name field
  • Add web url of your environment to URL field
  • Click on Save button to save information

Global Parameters

This is the area where you can define various global parameters to be used during your Test Script debugging mode. As you create your Test Script, you wanted to verify if its working as expected and for that you run it in debug mode. These parameters will be assign to test script debug mode.

  • Click on Settings
  • Click on Global Parameters
  • Click on Environment dropdown and select the environment on which you wanted to run all your test script debugging
  • Enter seconds in Timeout field to release the script if it hangs or fail - @Rajesh confirm and lets rename field to reflect what it’s doing , here and in TestBot à Run
  • Enter seconds in Wait for Element Timeout field to release the script if it cannot find the element within given time
  • Click on Save button to save the details

Custom Properties

If you have custom fields like a database needs to be connected and you wanted to pass it dynamically in test script then you can use the Custom Properties.

  • Click on Settings
  • Click on Global Parameters
  • Click on Add Custom Property
  • Enter name of the parameter in Name field and value of parameter in Value field
  • Click on Save button to save the details
